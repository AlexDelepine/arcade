Build Analysis

The Build Analysis offers a set of features to make build and pull request results more actionable.

Explore all the features offered by build analysis:

  • Build Analysis Website: Find the builds that are impacted by an error, that occurs either in tests or in timelines, across the dotnet organization, in the same way as Runfo does. You can also obtain the latest report for any known issue and a list of all open issues and their impact.
  • Build Analysis GitHub Check: Makes GitHub pull request results more actionable by highlighting build and test information most helpful to .NET developers.
  • Known Issues: Report known issues so these can be automatically identified and reported on failed builds.
  • Automatic Test Retries: Configure tests in your repo's pull requests to automatically retry when they encounter specific errors.
  • Automatic Build Retries: Configure your repo's pull request builds to automatically retry when they encounter specific errors.
  • Test Reporting Queries: We've collected your test results data into Kusto for ease of querying and put together a few queries to get started.
